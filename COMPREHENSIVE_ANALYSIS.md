# resume-agent 프로젝트 종합 분석 보고서

## 📋 프로젝트 개요

**resume-agent**는 Codex CLI 기반의 이력서 및 자기소개서 자동화 파이프라인입니다. 단순한 텍스트 생성이 아닌, 체계적이고 결정론적인 접근 방식을 통해 고품질의 취업 지원 서류를 작성하는 것을 목표로 합니다.

### 핵심 가치 제안
- **파이프라인 우선, LLM 보조**: 결정론적 엔진이 핵심 로직을 담당하고, Codex(LLM)은 보조 수단으로 활용
- **로컬 실행 가능**: 원격 LLM 없이도 독립적으로 작동하는 Codex-네이티브 애플리케이션
- **아티팩트 기반**: 모든 단계에서 내구성 있는 산출물 생성

## 🏗️ 아키텍처 분석

### 5계층 아키텍처

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 5: 아티팩트 계층 (Artifact Layer)                      │
│  - 마크다운, JSON 스냅샷, 유효성 검증 결과 생성                  │
├─────────────────────────────────────────────────────────────┤
│  Layer 4: 합성 계층 (Synthesis Layer)                         │
│  - 언어 합성이 필요한 부분에만 Codex 활용                       │
├─────────────────────────────────────────────────────────────┤
│  Layer 3: 결정론적 도메인 계층 (Deterministic Domain Layer)   │
│  - LLM에 의존하지 않는 핵심 의사결정 수행                      │
│  - questionClassifier, experienceAllocator, gapAnalyzer 등   │
├─────────────────────────────────────────────────────────────┤
│  Layer 2: 지식 계층 (Knowledge Layer)                         │
│  - 원시 저장소 + 패턴 KB로 참조 자료 정규화                     │
├─────────────────────────────────────────────────────────────┤
│  Layer 1: 입력 계층 (Input Layer)                             │
│  - 사용자 프로필, 경험 카드, 지원 프로젝트, 에세이 질문 입력      │
└─────────────────────────────────────────────────────────────┘
```

### 런타임 흐름
```
setup → crawl-base → my-profile → my-gaps → coach → writer → interview → export
```

### 3-Layer 설계 철학 (CODEX_CLI_ENHANCED)

| 계층 | 역할 | 설명 |
|------|------|------|
| **Layer 1: Deterministic Core** | 결정론적 핵심 | 질문 분류, 경험 배치, 갭 분석, 검증 등 |
| **Layer 2: Artifact Pipeline** | 산출물 파이프라인 | 각 단계가 durable markdown/json 산출물 생성 |
| **Layer 3: Codex Execution** | Codex 실행 | 합성 및 비판 단계만 담당 |

## 🎯 핵심 디자인 규칙

### 1. 결정론적 우선 (Deterministic First)
- 규칙으로 표현 가능한 결정은 LLM에 의존하지 않음
- 질문 분류, 경험 배치, 갭 분석 등은 deterministic 엔진으로 처리

### 2. 구조 재사용, 문구 복사 아님 (Structure, Not Copying)
- 참조 샘플은 수사적 구조와 증거 패턴만 참고
- 문구 직접 재사용 금지 (독특한 문구 재사용은 실패 위험으로 간주)

### 3. 모든 단계에서 아티팩트 작성 (Write Artifacts at Every Step)
- 각 단계가 독립적으로 검증 가능한 산출물 생성
- 마크다운 (사람용), JSON 스냅샷 (기계용), 유효성 검증 결과 (디버깅용)

### 4. 프롬프트 최소화 (Minimal Prompts)
- 유효성 검증된 스냅샷과 간결한 힌트만 사용
- 무제한 컨텍스트 대신 선택적 정보 전달

### 5. 검증 후 생성 (Validate After Generation)
- 모든 산출물은 생성 후 유효성 검증 수행

## 📦 주요 모듈 분석

### CLI 인터페이스 (cli.py)
주요 명령어 체계:

| 명령어 | 용도 | Codex 사용 여부 |
|--------|------|-----------------|
| `setup` | 초기 파일 및 구조 생성 | ❌ |
| `crawl-base` | 기초 데이터 수집 | ⚠️ 선택적 |
| `my-profile` | 프로필 정규화 | ⚠️ 선택적 |
| `my-gaps` | 갭 리포트 생성 | ❌ |
| `coach` | 경험 배치 및 설명 | ⚠️ 부분적 |
| `writer` | 답변 초안 작성 | ✅ 적극 활용 |
| `interview` | 면접 대비 전략 | ✅ 적극 활용 |
| `export` | 최종 산출물 조합 | ❌ |

### 데이터 모델 (models.py)
Pydantic 기반 데이터 모델:

**열거형(Enum)**:
- `CareerStage`: ENTRY, JUNIOR, MID, SENIOR
- `EvidenceLevel`: L1, L2, L3
- `QuestionType`: TYPE_A~TYPE_I (9가지 질문 유형)
- `VerificationStatus`: verified, needs_verification

**주요 모델**:
- `UserProfile`: 사용자 프로필 (이름, 경력 단계, 목표 직무 등)
- `Experience`: 경험 데이터 (STAR 기법 구조화)
- `ApplicationProject`: 지원 프로젝트 정보
- `KnowledgeSource`: 지식 소스 관리

## 🗄️ 지식베이스 (Knowledge Base) 설계

### 분리 원칙
지식베이스를 두 가지 저장소로 분리:

| 저장소 | 용도 | 특징 |
|--------|------|------|
| **원시 샘플 저장소** | 원본 전체 텍스트 보존 | 추적성과 오프라인 분석용 |
| **패턴 KB** | 압축된 메타데이터 | 코치/라이터/인터뷰 단계에서 검색용 |

### 분리 목적
1. 프롬프트 부풀림 방지
2. 의도치 않은 복사 방지
3. 검색 노이즈 감소

### 지원 소스 유형
- `local_markdown`, `local_text`, `local_csv_row`
- `user_url_public`, `manual_note`

### 검색 정책
- 라이터 단계 실행 시: 회사 별칭, 직무 별칭, 질문 유형, 추출된 질문 키워드로 쿼리
- 상위 3~5개 KB 엔트리 반환
- 프롬프트에는 간결한 요약만 전달 (대용량 원본 에세이는 기본적으로 전달하지 않음)

### 청소 규칙
- 플랫폼 마케팅 문구 제거
- 일반 CTA(Call-to-Action) 블록 제거
- 반복 빈 줄 정규화
- 질문 번호 유지
- 회사/직무 명명 유지
- 제목 분리 실패 시 파싱 신뢰도 표시

### 안전 규칙
- 소스 URL 보존
- 감사용 원본 소스 보존
- 라이터 단계에는 요약만 제공
- 독특한 문구 재사용은 실패 위험으로 간주

## 🔄 상태 관리 (State Management) 설계

### 상태 파일 구조
각 실행(run)은 독립적인 디렉토리에 상태를 저장:

```
my_run/
├── state.json          # 실행 상태 (진행률, 현재 단계 등)
├── profile.json        # 사용자 프로필 데이터
├── experiences.json    # 경험 카드 목록
├── application.json    # 지원 프로젝트 정보
├── gaps.json           # 갭 분석 결과
├── kb/                 # 지식베이스 디렉토리
│   ├── raw/            # 원시 샘플 저장소
│   └── patterns/       # 패턴 KB
├── outputs/            # 산출물 디렉토리
│   ├── coach/          # 코칭 결과
│   ├── writer/         # 작성 결과
│   ├── interview/      # 면접 준비 결과
│   └── export/         # 최종 내보내기
└── logs/               # 실행 로그
```

### 상태 전이
```
INITIALIZED → PROFILE_LOADED → GAPS_ANALYZED → COACHED → WRITTEN → INTERVIEWED → EXPORTED
```

## 📊 질문 유형 분류 체계 (QuestionType)

| 유형 | 코드 | 설명 | 예시 |
|------|------|------|------|
| 동기/지원동기 | TYPE_A | 왜 이 회사/직무에 지원했는지 | "지원동기를 작성해주세요" |
| 역량/경험 | TYPE_B | 핵심 역량과 관련 경험 | "가장 큰 성과를 설명해주세요" |
| 문제해결 | TYPE_C | 문제 해결 능력 | "어려움을 극복한 경험을 알려주세요" |
| 협업/팀워크 | TYPE_D | 협업 능력 | "팀 프로젝트 경험을 설명해주세요" |
| 성장/학습 | TYPE_E | 성장 마인드셋 | "실패에서 배운 점을 알려주세요" |
| 리더십 | TYPE_F | 리더십 경험 | "리더십을 발휘한 경험을 작성해주세요" |
| 직무이해 | TYPE_G | 직무에 대한 이해 | "이 직무의 핵심 역량은 무엇이라고 생각하나요" |
| 가치관/문화 | TYPE_H | 가치관 적합성 | "우리 회사의 가치관에 대해 어떻게 생각하시나요" |
| 자유/기타 | TYPE_I | 기타 자유 주제 | "자유롭게 자기소개를 해주세요" |

## 🚀 강점 및 특징 요약

### 기술적 강점
1. **결정론적 엔진**: LLM 의존도를 최소화하고 규칙 기반 로직으로 안정성 확보
2. **모듈化 설계**: 각 단계가 독립적으로 실행 및 검증 가능
3. **아티팩트 기반 워크플로우**: 모든 중간 결과가 파일로 저장되어 추적성 확보
4. **지식베이스 분리**: 원시 데이터와 패턴 데이터를 분리하여 효율적인 검색

### 비즈니스 가치
1. **일관된 품질**: 결정론적 엔진으로 일관된 출력 품질 보장
2. **시간 절약**: 수동 작성 대비 대폭적인 시간 절약
3. **맞춤형 결과**: 사용자 프로필과 경험에 기반한 개인화된 결과
4. **확장 가능**: 새로운 질문 유형이나 회사 추가가 용이

## 📝 결론

resume-agent는 단순한 텍스트 생성 도구를 넘어, **체계적인 취업 지원 프로세스를 자동화하는 파이프라인**입니다. 결정론적 엔진과 LLM의 조합으로 안정성과 창의성을 모두 확보했으며, 아티팩트 기반 워크플로우로 모든 단계의 추적성을 보장합니다.

특히 **"구조 재사용, 문구 복사 아님"**이라는 원칙은 단순한 복사붙여넣기가 아닌, 참조 자료의 구조적 패턴을 학습하여 독창적인 결과를 생성하는 접근 방식을 보여줍니다.

## 🔴 핵심 격차 분석 (Critical Gaps)

### 1. 의미적 이해 한계 (Semantic Understanding Gap)
**현재**: TF-IDF + 키워드 매칭 기반  
**목표**: 맥락적 의미 이해

| 문제 | 영향 |
|------|------|
| "고객 응대" ↔ "민원 처리" 의미적 유사성 감지 불가 | 경험-질문 매칭 부정확 |
| 경험의 **본질적 핵심 역량** 추출 불가 | 표면적 매칭에 그침 |
| 질문의 숨겨진 의도 파악 불가 | 표면적 답변 생성 |

**예시**:
```
현재 시스템: "고객 응대" 키워드 없으면 → 매칭 실패
상위 0.01%: "고객 응대" 경험의 본질("갈등 해결", "감정 관리", "압박 상황 대처") → 
             "민원 처리" 질문과 의미적 연결
```

### 2. 피드백 루프 부재 (Feedback Loop Absence)
**현재**: 추천 전략의 실제 합격 여부 추적 없음  
**목표**: 데이터 기반 전략 최적화

| 문제 | 영향 |
|------|------|
| "이 조합이 실제 합격했는가?" 검증 불가 | 추천 품질 개선 불가 |
| 합격/불합격 패턴 학습 없음 | 경험 배치 알고리즘 정체 |
| 피드백 기반 가중치 조정 없음 | 스코어링 로직 정적 |

### 3. 개인화 부재 (Personalization Gap)
**현재**: 모든 지원자에게 동일한 알고리즘 적용  
**목표**: 개인별 맞춤 전략

| 문제 | 영향 |
|------|------|
| 심리/커뮤니케이션 스타일 고려 없음 | 답변이 개인의 말투와 안맞음 |
| 강점/약점 개인 프로파일링 없음 | 차별화 전략 부재 |
| 자신감 수준 무시 | 부자연스러운 톤 생성 |

### 4. 실시간 데이터 부재 (Real-time Data Gap)
**현재**: 정적 데이터 (linkareer CSV)  
**목표**: 실시간 동향 반영

| 문제 | 영향 |
|------|------|
| 최신 채용 트렌드 반영 불가 | 시대적 적합성 부족 |
| 기업 뉴스/변화 감지 불가 | 정보 비대칭 |
| 내부 정보 접근 불가 | 구조적 한계 (불가피) |

## 🎯 우선순위 개선 로드맵

### Phase 1: 의미적 이해 강화 (1~2개월)
**목표**: 상위 5% → 상위 3%

| 개선사항 | 구현 사항 | 난이도 |
|----------|-----------|--------|
| 임베딩 기반 검색 | Ko-SBERT/multilingual-e5 + ChromaDB | 중간 |
| 형태소 분석 강화 | Kiwi/KoNLPy 통합 | 중간 |
| 경험 심층 분석기 | 핵심 역량/면접관 인상 예측 | 중간 |

### Phase 2: 결과 기반 학습 (2~3개월)
**목표**: 상위 3% → 상위 1%

| 개선사항 | 구현 사항 | 난이도 |
|----------|-----------|--------|
| 합격 결과 추적 | 지원 결과 입력 → 통계 분석 | 낮음 |
| 피드백 루프 강화 | 배치 × 결과 상관관계 학습 | 낮음 |
| A/B 테스트 프레임워크 | 전략 비교 및 최적화 | 중간 |

### Phase 3: 개인화 및 심층 코칭 (3~4개월)
**목표**: 상위 1% → 상위 0.1%

| 개선사항 | 구현 사항 | 난이도 |
|----------|-----------|--------|
| 지원자 프로파일링 | 글쓰기 스타일, 강점/약점 분석 | 높음 |
| 기업별 심층 분석 | 미션/values 파싱, 합격 패턴 DB | 높음 |
| 면접 심리 코칭 | 긴장 관리, 자신감 구축 가이드 | 중간 |

### Phase 4: 실시간 데이터 통합 (4~6개월)
**목표**: 상위 0.1% → 상위 0.01%

| 개선사항 | 구현 사항 | 난이도 |
|----------|-----------|--------|
| 최신 동향 수집 | 채용 페이지 변경 감지 | 높음 |
| 커뮤니티 데이터 | 합격/불합격 사례 공유 | 높음 |
| 내부 정보 대체 | 공개된 내부자 인터뷰 분석 | 높음 |
